# Moneyraham Homepage Design

> 초보자를 위한 한국어 홈페이지 제작 Codex 스킬

## 저작권 및 개량 안내

- **원본 UI/UX 디자인 인텔리전스 저작권자:** Next Level Builder — [UI/UX Pro Max](https://github.com/nextlevelbuilder/ui-ux-pro-max-skill), Copyright (c) 2024 Next Level Builder, MIT License
- **이 수강생용 스킬의 개량·배포자:** Moneyraham / GitHub [`uncheon0912`](https://github.com/uncheon0912), Copyright (c) 2026

`Moneyraham_homepage_Disign`은 UI/UX Pro Max의 디자인 문제 해결 방식에서 영감을 받아, **한국어 홈페이지 제작 흐름**, **초보자용 설치 안내**, **반응형·접근성 검증**, **History 기록 방식**을 추가하여 개량한 학습용 스킬입니다.

이 저장소는 원본 UI/UX Pro Max의 공식 저장소나 공식 제휴 프로젝트가 아닙니다. 원본 코드·데이터·문서를 복사하거나 수정해 다시 배포할 경우에는 원본의 MIT 저작권 및 라이선스 고지를 반드시 함께 유지해야 합니다.

## 1. 이 스킬은 무엇인가요?

Codex에게 “홈페이지를 만들어줘”라고 요청할 때, 단순한 예쁜 화면만 제안하지 않도록 돕는 작업 가이드입니다. 방문자가 이해할 메시지, 신뢰 요소, 상담/구매 같은 행동 버튼(CTA), 모바일 화면, 접근성, 실제 미리보기 확인까지 순서대로 챙기게 합니다.

이 스킬은 특히 다음과 같은 경우에 유용합니다.

- 처음 홈페이지를 만드는 개인 사업자·수강생
- 한국어 브랜드 소개 페이지, 랜딩 페이지, 포트폴리오 사이트
- 상담 신청, 예약, 상품 문의, 뉴스레터 구독이 필요한 페이지
- 기존 홈페이지의 디자인·문구·모바일 화면을 개선하고 싶은 경우

### 스킬이 해주는 일

1. 브랜드, 고객, 목표 행동을 바탕으로 페이지 구조를 잡습니다.
2. 첫 화면의 핵심 메시지와 CTA를 우선 설계합니다.
3. 색상, 글꼴, 여백, 카드, 버튼 등 일관된 디자인 기준을 정합니다.
4. 데스크톱과 모바일 화면, 키보드 조작, 글자 대비를 확인합니다.
5. 실제로 실행되는 미리보기를 확인하고 작업 기록을 `History`에 남깁니다.

### 스킬이 대신 결정할 수 없는 일

- 실제 사업 정보, 가격, 연락처, 약관, 로고의 진위
- 근거 없는 후기·수상·매출·인증 문구
- 사용자 승인 없는 외부 공개, 도메인 연결, 결제 설정

이런 정보는 반드시 본인이 제공하거나 확인해야 합니다.

## 3. Windows에서 설치하기

### 1단계 — 터미널 열기

Codex에서 새 작업을 열고 터미널을 열거나, Windows PowerShell을 실행하세요.

### 2단계 — 아래 명령을 통째로 복사해 실행하기

```powershell
python "$env:USERPROFILE\.codex\skills\.system\skill-installer\scripts\install-skill-from-github.py" `
  --repo uncheon0912/bujasaram-homepage-design `
  --path skills/moneyraham-homepage-design
```

성공하면 아래와 비슷한 문구가 보입니다.

```text
Installed moneyraham-homepage-design to ...\skills\moneyraham-homepage-design
```

### 3단계 — Codex 새 작업 열기

설치가 끝난 뒤 Codex에서 새 작업을 열어 주세요. 이전에 열어 둔 작업에서는 새 스킬이 바로 보이지 않을 수 있습니다.

### 4단계 — 설치 확인하기

새 작업에서 아래 문장을 입력해 보세요.

```text
$moneyraham-homepage-design을 사용해서 상담 신청이 가능한 한국어 반응형 홈페이지를 만들어줘.
```

스킬 이름이 보이지 않더라도, 다음처럼 홈페이지 제작을 구체적으로 요청할 수 있습니다.

```text
30~40대 자영업자를 위한 세무 상담 홈페이지를 만들어줘.
첫 화면에는 상담 신청 버튼을 넣고, 모바일에서도 보기 좋게 만들어줘.
```

명시 호출인 `$moneyraham-homepage-design` 방식이 가장 확실합니다.

## 4. 처음 홈페이지를 만들 때 쓰는 요청 예시

아래 문장을 복사한 뒤, 대괄호 안만 자신의 정보로 바꾸세요.

```text
$moneyraham-homepage-design을 사용해서 [브랜드명] 홈페이지를 만들어줘.
[무엇을 판매하거나 제공하는지]를 처음 방문한 사람이 쉽게 이해하게 해줘.
첫 화면, 서비스 소개, 고객이 얻는 장점, 문의하기, 푸터를 넣어줘.
```

### 예시 B — 상담 신청 랜딩 페이지

```text
$moneyraham-homepage-design을 사용해서 [서비스명] 상담 신청 랜딩 페이지를 만들어줘.
주 고객은 [고객층]이고, 가장 중요한 행동은 [상담 신청/예약/문의]이야.
신뢰감 있는 한국어 디자인으로 만들고 모바일 화면도 확인해줘.
```

### 예시 C — 예약 홈페이지

```text
$moneyraham-homepage-design을 사용해서 [업종] 예약 홈페이지를 만들어줘.
```

### 예시 D — 기존 홈페이지 개선

```text

수정 전 변경 계획을 먼저 보여주고 승인 후 작업해줘.
```

### 예시 E — 포트폴리오 홈페이지

```text
$moneyraham-homepage-design을 사용해서 [직업/분야] 포트폴리오 홈페이지를 만들어줘.
프로필, 대표 작업, 작업 방식, 문의 CTA를 넣고 차분하고 전문적인 분위기로 구성해줘.
```

## 5. 홈페이지가 만들어지는 순서

이 스킬은 아래 순서를 권장합니다.

1. **목표 확인** — 누구를 위한 홈페이지인지, 방문자가 어떤 행동을 해야 하는지 정합니다.
2. **구성 제안** — 첫 화면, 서비스, 신뢰 요소, CTA 등 페이지 구성을 제안받습니다.
3. **승인** — 여러 파일을 만들거나 기존 파일을 바꾸기 전 계획을 확인합니다.
4. **구현** — 실제 HTML/React 등 프로젝트 기술에 맞는 홈페이지를 만듭니다.
5. **미리보기 검증** — 실행 화면에서 메뉴, 버튼, 모바일 화면이 동작하는지 확인합니다.
6. **History 기록** — 결정 이유, 수정 파일, 실행 방법, 검증 결과를 프로젝트의 `History` 폴더에 기록합니다.

`History`는 나중에 잘못된 요청으로 디자인이 바뀌었을 때 이전 결정을 확인하고 복원하는 데 도움이 됩니다.

## 6. 초보자 체크리스트

홈페이지를 받기 전에 아래를 확인하세요.

- [ ] 첫 화면만 봐도 무엇을 제공하는지 알 수 있다.
- [ ] 방문자가 눌러야 할 주요 버튼이 한눈에 보인다.
- [ ] 연락처·예약·문의 정보가 실제 정보인지 확인했다.
- [ ] 휴대폰 화면에서도 글자와 버튼이 충분히 크다.
- [ ] 키보드로 버튼과 메뉴를 이동할 수 있다.
- [ ] 이미지, 후기, 인증, 가격에 사실과 다른 내용이 없다.
- [ ] 실제 미리보기에서 메뉴와 CTA 버튼을 눌러 봤다.
- [ ] `History`에 작업 기록이 남아 있다.

## 7. 자주 묻는 질문

### Q. `$moneyraham-homepage-design`이 보이지 않아요.

설치 후 Codex 새 작업을 열었는지 먼저 확인하세요. 계속 보이지 않으면 터미널에서 설치 명령을 다시 실행하기 전에, 같은 이름의 기존 스킬 폴더가 있는지 확인하고 중요한 개인 수정 내용을 백업하세요.

### Q. `python`을 찾을 수 없다고 나와요.

Python 3을 설치한 뒤 터미널을 다시 열어 `python --version`이 표시되는지 확인하세요.

### Q. 설치 도구 파일을 찾을 수 없다고 나와요.

Codex를 최신 상태로 열었는지 확인하고, 아래 경로가 존재하는지 점검하세요.

```powershell
Test-Path "$env:USERPROFILE\.codex\skills\.system\skill-installer\scripts\install-skill-from-github.py"
```

`False`가 나오면 Codex 환경에 기본 설치 도구가 준비되지 않은 상태입니다. Codex를 업데이트하거나 강의 안내에 따라 수동 설치 방법을 받아 진행하세요.

### Q. 다른 프로젝트에서도 쓸 수 있나요?

네. 위 설치 방식은 사용자 Codex의 스킬 폴더에 설치하므로, 새 프로젝트에서도 사용할 수 있습니다. 프로젝트별로만 쓰고 싶다면 해당 프로젝트의 `.codex/skills/`에 설치하는 방식을 사용하세요.

### Q. 스킬을 업데이트하려면 어떻게 하나요?

기존 폴더에 개인적으로 수정한 내용이 있다면 먼저 백업하세요. 이후 새 버전의 설치·업데이트 방법은 이 저장소의 안내 또는 강의 공지를 따르세요. 백업 없이 기존 스킬 폴더를 삭제하거나 덮어쓰지 마세요.
